1. Project Overview

Suraksha.AI is an AI-powered cybersecurity web application designed to detect phishing attacks, malicious files, and suspicious digital behavior. The platform integrates Machine Learning with Generative AI to provide accurate threat detection along with clear, human-readable explanations that help users understand why a threat exists, not just whether it exists.

Objectives

  • Detect phishing URLs, emails, and digital messages
  • Analyze uploaded files for malware and hidden threats
  • Provide AI-generated explanations for every detection result
  • Enable non-technical users to understand cybersecurity risks
  • Support multilingual explanations for global accessibility
  • Promote cybersecurity awareness through explainable AI

4. Working Model

Phishing Detection

  1. User submits a URL, email, or message
  2. Input is cleaned and preprocessed
  3. Machine learning model predicts threat probability
  4. Gemini AI generates a clear explanation
  5. Explanation is displayed in the selected language

Malware Detection

  1. User uploads a file
  2. File metadata and behavioral patterns are analyzed
  3. ML model classifies the file as safe or malicious
  4. AI provides safety advice and next steps

5. Vault Analysis

Vault Analysis is a secure intelligence module that stores historical scan results, enabling users to review previous threats, analyze trends, and regenerate explanations as threat intelligence evolves.

Features

  • Encrypted storage of phishing and malware scan history
  • Threat severity and trend analysis
  • AI-powered re-explanation of historical scans
  • Improved and updated security recommendations

Workflow

  1. Scan result is securely stored in the vault
  2. User selects a previous scan
  3. AI regenerates an updated explanation
  4. Refined safety guidance is displayed

6. Multilingual AI Explanations

Suraksha.AI delivers multilingual explanations using Generative AI, ensuring cybersecurity insights are accessible to users across different regions and languages.

Covered Features

9. API Documentation

POST /analyze-phishing
Input: URL or text
Output: threat_level, confidence_score, explanation

POST /scan-file
Input: file upload
Output: risk_status, recommendations

POST /vault/history
Output: list of previous scans

POST /explain?lang=hi
Output: translated explanation
